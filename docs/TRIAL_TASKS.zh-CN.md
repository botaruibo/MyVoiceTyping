# 30 秒试用任务 / Trial tasks

这页给第一次打开 MyVoiceTyping 的用户：不用想测试语句，直接复制下面的场景，按住快捷键说一遍，看输出是否适合你。

建议先在“备忘录”或普通文本框里测试，不要一开始就在真实工作群、客户文档或 AI agent 输入框里测试。

> Safety：下面都是虚构示例。请不要用真实公司机密、客户信息、Token、密码、私聊原文或内部代码测试公开反馈。

## 怎么测试？

1. 打开一个普通文本框；
2. 按住 MyVoiceTyping 快捷键；
3. 口述下面任意一条“测试口述”；
4. 松开快捷键；
5. 看输出是否被粘贴到当前输入位置；
6. 检查错字、标点、断句、技术词和原意是否保留。

如果某个任务对你特别重要，可以把脱敏后的结果提交到 [使用场景反馈 / Use case feedback](https://github.com/botaruibo/MyVoiceTyping/issues/new/choose)。

## 任务 1：AI Coding prompt

适合：Codex、Claude Code、Cursor、Copilot、Qwen Code 用户。

测试口述：

```text
帮我检查一下登录页面，用户点击发送验证码以后按钮应该进入倒计时，如果接口报错需要恢复按钮并显示错误信息。
```

理想输出：

```text
帮我检查一下登录页面：用户点击“发送验证码”以后，按钮应该进入倒计时；如果接口报错，需要恢复按钮并显示错误信息。
```

观察点：

- “登录”“验证码”“倒计时”是否正确；
- 是否自动补了冒号、引号、分号；
- 是否适合直接作为 AI Coding prompt 的第一稿；
- 是否还有机会在发送前手动 review。

## 任务 2：工作消息 / 邮件

适合：飞书、微信、Slack、邮件、团队协作工具。

测试口述：

```text
这个需求我看了一下，主要风险是权限边界和数据同步，如果这周要上线的话建议先做灰度。
```

理想输出：

```text
这个需求我看了一下，主要风险是权限边界和数据同步。如果这周要上线，建议先做灰度。
```

观察点：

- 是否把长句断开；
- 是否保持原意；
- 润写是否过度正式；
- 是否适合直接发给同事。

## 任务 3：中文 / 中英混合技术词

适合：开发者、产品经理、AI Coding 用户。

测试口述：

```text
把这个 login form 的 submit handler 改一下，如果 response code 是四零一就跳登录页。
```

理想输出：

```text
把这个 login form 的 submit handler 改一下，如果 response code 是 401，就跳登录页。
```

观察点：

- `login form`、`submit handler`、`response code` 是否保留；
- “四零一”是否能整理成 `401`；
- 中英文之间空格是否自然；
- 是否误改技术含义。

## 任务 4：Bug 复现说明

适合：GitHub Issue、PR 评论、团队 IM。

测试口述：

```text
复现步骤是先打开设置页面，然后切换模型，再点保存，这个时候页面会卡住，控制台没有报错，但是按钮一直是 loading。
```

理想输出：

```text
复现步骤是：先打开设置页面，然后切换模型，再点保存。这个时候页面会卡住，控制台没有报错，但是按钮一直是 loading。
```

观察点：

- 是否能分出复现步骤和现象；
- `loading` 是否保留；
- 是否适合粘贴到 Issue 里；
- 是否出现删信息或改原意。

## 任务 5：隐私敏感输入的安全测试

适合：在意本地优先、不想默认把内容交给云端语音输入的人。

测试口述：

```text
请帮我整理一段虚构的客户需求，客户希望下个月上线一个报表页面，需要支持导出和权限控制。
```

理想输出：

```text
请帮我整理一段虚构的客户需求：客户希望下个月上线一个报表页面，需要支持导出和权限控制。
```

观察点：

- 请只用虚构内容测试；
- 输出是否留在当前输入框，方便你确认后再发送；
- 是否符合你对隐私敏感输入的安全感；
- 如果你要反馈隐私问题，请先阅读 [Privacy / Data Safety](./PRIVACY.md)。

## 任务 6：Self-evaluation / 自进化样例

适合：想让工具未来更懂自己写作习惯的人。

测试口述：

```text
这个接口这里感觉有点问题，帮我改一下。
```

你可能会手动改成：

```text
这个接口的错误处理逻辑可能有问题，帮我检查并补充异常分支。
```

观察点：

- 初始输出和你最终想要的文本差异在哪里；
- 是语气问题、术语问题，还是信息不够明确；
- 这类“初始输出 → 用户确认文本”未来可以作为本地偏好样本；
- 自进化数据默认应本地保存、用户可控、不自动上传。

更多说明：[Self-evaluation / 本地自进化说明](./SELF_EVALUATION.md)。
