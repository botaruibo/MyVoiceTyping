# Demo / 使用效果示例

这份文档用几个典型场景展示 MyVoiceTyping 想解决的问题：不是单纯“录音转文字”，而是把口述内容变成可以直接粘贴到当前输入框里的可用文本。

> 说明：下面是示例文本，用于展示目标体验。真实输出会受麦克风、环境噪音、语速、模型版本和本地配置影响。

如果你只是想先快速判断项目是否值得试，可以先看：[30 秒 Demo walkthrough](./DEMO_30S.zh-CN.md)。

如果你想录制或转发一个 30–60 秒演示视频，可以参考：[60 秒 Demo 录制脚本](./DEMO_VIDEO.md)。当前 Demo/GIF 制作和回填进展追踪见：[Issue #8](https://github.com/botaruibo/MyVoiceTyping/issues/8)。

## 核心体验

```text
按住快捷键 → 说话 → 松开 → 本地 ASR → 标点恢复 → 轻量纠错/润写 → 粘贴
```

MyVoiceTyping 更适合日常输入场景：

- 写需求；
- 写 PR / Issue；
- 给 AI Coding 工具描述任务；
- 回消息、写邮件；
- 记录想法；
- 需要尽量本地处理、不希望音频和文本上传云端的场景。

## 示例 1：普通中文输入

### 口述内容

```text
今天下午我想整理一下这个语音输入项目的 readme 然后补一下安装步骤和常见问题
```

### 可能的 ASR 粗文本

```text
今天下五我想整理一下这个语音输入项目的read me然后补一下安装步骤和常见问题
```

### MyVoiceTyping 目标输出

```text
今天下午我想整理一下这个语音输入项目的 README，然后补一下安装步骤和常见问题。
```

重点：

- 修正“下五 → 下午”；
- 保留 README 这类技术词；
- 补充标点和断句；
- 不改变原意。

## 示例 2：AI Coding 需求描述

### 口述内容

```text
帮我看一下这个登录页面的问题 用户点击发送验证码以后按钮应该进入倒计时状态 如果接口报错需要恢复按钮并显示错误信息
```

### 可能的 ASR 粗文本

```text
帮我看一下这个登陆页面的问题用户点击发送验证码以后按钮应该进入倒记时状态如果接口报错需要恢复按钮并显示错误信息
```

### MyVoiceTyping 目标输出

```text
帮我看一下这个登录页面的问题：用户点击“发送验证码”以后，按钮应该进入倒计时状态；如果接口报错，需要恢复按钮并显示错误信息。
```

重点：

- 修正“登陆 → 登录”；
- 修正“倒记时 → 倒计时”；
- 自动加引号、冒号、分号；
- 让口述内容变成更适合直接发给 AI Coding 工具的任务描述。

```text
复现步骤是先打开设置页面 然后切换模型 再点保存 这个时候页面会卡住 控制台没有报错 但是按钮一直是 loading
```

### 可能的 ASR 粗文本

```text
复现步骤是先打开设置页面然后切换模型再点保存这个时候页面会卡住控制台没有报错但是按钮一直是loading
```

### MyVoiceTyping 目标输出

```text
复现步骤是：先打开设置页面，然后切换模型，再点保存。这个时候页面会卡住，控制台没有报错，但是按钮一直是 loading。
```

- 自动断句；
- 保留 `loading`；
- 更适合直接粘贴到 GitHub Issue、PR 评论或团队 IM。

## 示例 4：隐私敏感的工作输入

如果你经常口述公司需求、内部项目名、Bug 细节或私人消息，云端语音输入工具可能会让人有顾虑。

MyVoiceTyping 的路线是本地优先：

- 音频尽量在本机处理；
- ASR 后文本尽量在本机纠错/润写；
- 默认不把音频和文本上传云端；
- 模型、数据集、应用代码都公开，方便审查和二次开发。

更完整的数据边界见：

- [Privacy / Data Safety](./PRIVACY.md)

## 示例 5：Self-evaluation / 自进化

语音输入的长期难点不是只把字识别出来，而是越来越贴合“你会怎么写”。

比如：

### 初始输出

```text
这个接口这里感觉有点问题帮我改一下
```

### 用户最终确认文本

```text
这个接口的错误处理逻辑可能有问题，帮我检查并补充异常分支。
```

如果用户愿意保留这类“初始输出 → 用户确认文本”的本地样例，alpha-0.03 开始会写入本机 `voice_history.jsonl`，先用于本地评估、热词和规则优化；在用户确认后，可参与本机 MLX LoRA 调优，并转换为 GGUF Q4 本地模型，让模型逐渐学会：

- 你的常用词；
- 你的语气；
- 你的项目术语；
- 你的任务描述方式；
- 你希望口语被整理成什么样。

这类数据可能包含真实工作内容和个人表达习惯，因此默认原则应该是本地保存、用户可控、不自动上传云端，也不自动进入公开数据集。

## 如何试用

1. 下载最新 Release：
   - [MyVoiceTyping Releases](https://github.com/botaruibo/MyVoiceTyping/releases)
2. 安装并授予 macOS 权限：
   - 麦克风；
   - 辅助功能；
   - 输入监控。
3. 找一个普通输入框，比如备忘录、浏览器文本框或编辑器。
4. 按住快捷键说一句中文，松开后查看结果。
