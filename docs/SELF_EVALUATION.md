# Self-evaluation / 本地自进化说明

MyVoiceTyping 从 alpha-0.03 开始把 **Self-Evolution / 本地自进化** 作为重点能力推进：它不是只做一次性的“语音转文字”，而是让本地模型逐步理解用户自己的表达习惯。

简单说：用户每次语音输入后，系统会得到一个初始结果；如果用户把它改成了更满意的版本，这个“初始输出 → 用户确认文本”的差异，就可以成为本机 `voice_history.jsonl` 里的本地偏好样本。后续这些样本可以用于评估、词库更新、规则优化，也可以在用户确认后参与本机 MLX LoRA 轻量调优，并转换成 GGUF Q4 模型替换本地文本改写模型。

## 想解决什么问题？

通用语音输入和通用润写模型不一定理解：

- 你的常用词；
- 项目名、公司内部术语、技术缩写；
- 中英混合表达；
- 你偏好的语气和句式；
- AI Coding prompt 的写法；
- 你习惯保留或删除的口语词。

Self-evaluation / 自进化想做的是：让模型通过用户最终确认的文本，逐步学习“你真正想要的输出长什么样”。

## 一个样例长什么样？

本地样例可以采用类似结构：

```json
{
  "version": 1,
  "created_at": "2026-07-17T03:47:38+08:00",
  "source": "local_user_confirmed_edit",
  "input_context": {
    "app": "editor",
    "language": "zh-CN",
    "scenario": "ai_coding_prompt"
  },
  "raw_transcript": "帮我看一下这个函数为什么会报错然后给一个修复方案",
  "model_output": "帮我看一下这个函数为什么会报错，然后给一个修复方案。",
  "user_confirmed_text": "帮我分析这个函数为什么会报错，并给出一个最小改动的修复方案。",
  "user_action": "edited_and_confirmed",
  "training_allowed": false,
  "tags": ["punctuation", "rewrite", "ai-coding"]
}
```

这里最重要的不是字段名字，而是边界：

- 样例来自用户确认后的结果；
- 用户能决定是否保存；
- 用户能决定是否参与本地训练；
- 默认不应该上传云端；
- 不应该自动进入公开数据集。

## alpha-0.03 的本地自进化链路

alpha-0.03 的核心链路是：

1. 在本机记录原始转写、模型输出、用户最终确认文本和输入场景；
2. 将这些样例写入本机 `voice_history.jsonl`；
3. 训练前让用户确认哪些样例可以参与本地调优；
4. 使用本机 MLX LoRA 调优本地文本改写模型；
5. 调优完成后转换为 GGUF Q4；
6. 通过完整性检查后，再替换正式使用的本地模型。

这个方向的目标是让模型逐步更懂用户自己的技术词、项目名、中英混合习惯和表达偏好，也就是“越用越顺手”。

## 可以怎么用？

短期更适合做这些本地优化：

1. 评估模型输出是否越来越接近用户确认文本；
2. 提取常用词、项目名、技术词，生成本地热词或偏好词表；
3. 总结用户常见修改习惯，例如“更正式一点”“保留英文术语”“删除口头禅”；
4. 生成本地规则或 prompt 偏好；
5. 在用户明确授权后，用本机样例做本地小模型轻量调优。

后续还需要继续打磨：

- 本地样例管理界面；
- 样例预览、搜索、删除、导出；
- 本地训练前的敏感信息扫描；
- 训练前后效果对比；
- 模型替换前后的回滚流程；
- 多个场景的偏好配置，例如聊天、文档、AI Coding、会议纪要。

## 数据安全边界

Self-evaluation 数据往往比普通日志更敏感，因为它可能包含真实工作内容、客户上下文、代码需求、私人消息和个人写作习惯。

因此 MyVoiceTyping 的原则是：

- 默认本地优先；
- 不默认上传；
- 不把用户真实私有样例提交到公开仓库；
- 不把用户隐私样例合并进公开数据集；
- 不在用户不知情的情况下用于云端训练；
- 让用户能保存、删除、导出，并决定是否参与本地训练。

如果未来加入云端同步、云端训练或共享样例能力，必须明确说明数据去向，并提供显式开关。

## 当前完成度

当前 self-evaluation / 自进化已经进入 alpha-0.03 的重点建设范围，但仍不应理解为“完整成熟的自动训练闭环已经全部完成”。

当前已经公开的基础资产包括：

- App：[MyVoiceTyping](https://github.com/botaruibo/MyVoiceTyping)
- 本地文本润写模型：[MyVoiceTyping-1.5b-q4](https://modelscope.cn/models/botaruibo/MyVoiceTyping-1.5b-q4)
- ASR 后纠错/润写数据集：[MyVoiceTyping-Dataset](https://github.com/botaruibo/MyVoiceTyping-Dataset)

下一步更现实的推进顺序是：

1. 继续完善本地样例管理；
2. 继续完善保存、删除、导出和训练授权边界；
3. 支持用户预览和清理样例；
4. 完善本地 MLX LoRA 调优、GGUF Q4 转换和完整性检查；
5. 增加训练前后效果对比和模型回滚流程。

## 公开反馈时请注意

如果你想帮助改进 self-evaluation 方向，欢迎提交可以公开的样例，但请不要上传包含以下内容的原始音频或文本：

- 公司机密；
- 客户信息；
- Token、密码、密钥；
- 财务、医疗、法律等敏感内容；
- 私人聊天记录；
- 不适合公开的代码或业务上下文。

更安全的反馈方式是：把真实内容替换成虚构内容，只保留“原始转写 → 期望文本”的结构和问题类型。
