# 使用场景 / Use cases

这页帮助新用户快速判断：MyVoiceTyping 是否适合自己的输入场景。

MyVoiceTyping 不是会议转录软件，也不是完整商业输入法。它更像一个本地优先的 macOS 中文语音输入层：按住快捷键说话，松开后转写、补标点、轻量纠错/润写，再把结果粘贴到当前输入框。

## 1. AI Coding prompt

适合：

- 给 Codex、Claude Code、Cursor、Copilot、Qwen Code 描述需求；
- 口述 Bug 复现步骤；
- 口述重构计划；
- 口述 PR 描述、commit message、测试用例；
- 把脑子里的长需求先说出来，再手动检查后发送。

示例口述：

```text
帮我检查一下登录页面，用户点击发送验证码以后按钮应该进入倒计时，如果接口报错需要恢复按钮并显示错误信息。
```

目标输出：

```text
帮我检查一下登录页面：用户点击“发送验证码”以后，按钮应该进入倒计时；如果接口报错，需要恢复按钮并显示错误信息。
```

为什么适合 MyVoiceTyping：

- 技术需求通常很长，用说的比打字快；
- 结果先进入输入框，用户确认后再发送，不会直接触发错误 agent action；
- 中文 / 中英混合表达更常见，需要 ASR 后补标点、保留技术词、轻量整理。

相关反馈入口：

- [AI Coding prompt 反馈](https://github.com/botaruibo/MyVoiceTyping/issues/5)
- [60 秒 Demo 脚本](./DEMO_VIDEO.md)

## 2. 工作消息、文档和邮件

适合：

- 长 IM 回复；
- 邮件初稿；
- 产品需求说明；
- 周报、日报、会议纪要摘要；
- 把零散想法先说成一段可编辑文本。

示例口述：

```text
这个需求我看了一下，主要风险是权限边界和数据同步，如果这周要上线的话建议先做灰度。
```

目标输出：

```text
这个需求我看了一下，主要风险是权限边界和数据同步。如果这周要上线，建议先做灰度。
```

为什么适合 MyVoiceTyping：

- 中文工作表达通常需要标点和断句；
- 原始转写不一定能直接发，需要轻量纠错和润写；
- 文本会先落到当前输入位置，方便用户最后检查。

## 3. 隐私敏感输入

适合：

- 公司需求；
- 客户上下文；
- 代码思路；
- 私人消息；
- 不希望默认交给云端语音输入的内容。

MyVoiceTyping 的方向是本地优先：默认尽量让音频、原始转写、润写后的文本留在本机流程里，减少敏感内容外传风险。

注意边界：

- “本地优先”不等于“永远不会联网”；
- 首次模型下载、用户手动打开外部链接等场景仍可能访问网络；
- 不要把真实公司机密、Token、密码、客户信息提交到公开 issue、评论区或数据集。

相关说明：

- [Privacy / Data Safety](./PRIVACY.md)
- [Security Policy](../SECURITY.md)

## 4. 写作者和中文长文本输入

适合：

- 想先口述一段想法；
- 写文章提纲；
- 写评论、笔记、读书摘录；
- 把“想到什么说什么”整理成更清楚的句子。

MyVoiceTyping 的重点不是把用户变成播音员，而是允许用户用自然口语输入，再通过标点恢复、常见错字修正和轻量润写，变成更适合阅读的文字。

如果你写的是正式发布内容，建议把 MyVoiceTyping 当成“第一稿输入工具”，最后仍由你自己审阅。

## 5. 中文 / 中英混合输入

适合：

- 中文夹英文术语；
- 技术词、产品名、变量名；
- 口述 prompt、issue、PR、接口说明；
- 不想频繁切换中英文键盘。

示例：

```text
把这个 login form 的 submit handler 改一下，如果 response code 是四零一就跳登录页。
```

这类内容很容易在原始 ASR 阶段出现大小写、断句、术语识别问题，因此 MyVoiceTyping 会把“ASR 后处理”作为重点：标点、错词、轻量润写和未来的本地偏好样本。

## 6. Self-evaluation / 本地自进化

适合：

- 你希望工具慢慢理解自己的常用词；
- 你有固定项目名、团队黑话、技术词；
- 你希望模型逐渐贴近自己的写作语气；
- 你愿意在本地保留“初始输出 → 用户确认文本”的偏好样本。

这个方向目前是产品路线，不应理解为完整自动训练闭环已经全部完成。

原则：

- 用户确认后的样本才有价值；
- 默认不上传云端；
- 不自动进入公开数据集；
- 用户应能保存、删除、导出，并决定是否参与本地训练。

相关说明：

- [Self-evaluation / 本地自进化说明](./SELF_EVALUATION.md)

## 什么时候不适合？

MyVoiceTyping 可能不适合这些场景：

- 你需要 iOS / Android / Windows / Linux 支持；
- 你需要成熟商业客服、团队管理、企业 SLA；
- 你主要做英文长文听写；
- 你想要会议录音转写、多人说话分离、字幕生成；
- 你不想处理 macOS 权限、本地模型下载或开源项目早期问题；
- 你需要“识别后自动发送”，而不是“先粘贴、再确认”。

如果你追求最成熟的商业体验，可以继续比较 Typeless、闪电说、Typeoff、Wispr Flow 等工具。  
如果你更在意开源、本地优先、0 费用、中文 / 中英混合输入和可改造性，可以试试 MyVoiceTyping。

## 推荐开始方式

1. 先看 [Quickstart / macOS 试用指南](./QUICKSTART.zh-CN.md)；
2. 下载最新 [Release](https://github.com/botaruibo/MyVoiceTyping/releases)；
3. 用一个不含敏感信息的句子试 30 秒；
4. 如果适合，再试 AI Coding prompt 或工作消息；
5. 如果遇到问题，使用 [Issue templates](https://github.com/botaruibo/MyVoiceTyping/issues/new/choose) 反馈；
6. 如果你只是想反馈“这个工具是否适合我的场景”，可以选择 `使用场景反馈 / Use case feedback` 模板。
