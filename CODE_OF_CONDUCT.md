# Code of Conduct / 社区行为准则

MyVoiceTyping 希望成为一个友好、克制、重视隐私和真实反馈的开源项目。无论你是来试用、提 bug、提交样例、讨论 Typeless / 闪电说 / Typeoff 对比，还是贡献代码和文档，都请遵守下面的基本原则。

## 我们欢迎的行为

- 尊重不同经验水平的用户和贡献者；
- 用可复现事实讨论问题，例如环境、步骤、实际输出、期望输出；
- 在反馈语音输入、ASR、润写或 self-evaluation 问题时，优先使用虚构或脱敏样例；
- 对竞品和同类工具保持克制，不使用“吊打”“完爆”等攻击性表达；
- 承认项目仍在早期，帮助指出真实缺口，而不是要求一次性完成所有平台和功能；
- 在讨论隐私、安全、日志、模型下载或训练数据时，尽量提供最小可复现信息。

## 不接受的行为

- 人身攻击、骚扰、威胁、歧视或持续挑衅；
- 发布他人的隐私、公司机密、客户信息、Token、密码、私聊原文、真实内部代码或未授权音频；
- 在 Issue、Discussion、PR 或评论中反复刷屏、无上下文推广或引战；
- 故意提交误导性安全报告、伪造样例或恶意夸大项目能力；
- 要求维护者默认收集、上传或公开用户隐私数据；
- 在没有上下文的情况下攻击 Typeless、闪电说、Typeoff、Wispr Flow 或其他同类项目。

## 隐私与样例提交

这个项目涉及语音、文本、日志和 self-evaluation 数据，反馈时请特别注意：

- 不上传真实敏感音频；
- 不贴真实客户、公司、个人或私聊内容；
- 不贴 Token、密码、API key、内部 URL、私有仓库路径；
- 如果要提交“原始转写 → 期望文本”样例，请先脱敏或构造等价虚构样例；
- 如果你发现安全或隐私问题，请先阅读 [Security Policy](SECURITY.md) 和 [Privacy / Data Safety](docs/PRIVACY.md)。

## 处理方式

维护者可以视情况：

- 提醒编辑或删除包含敏感信息的内容；
- 关闭重复、无关、攻击性或无法复现的问题；
- 隐藏或删除骚扰、刷屏、引战或泄露隐私的评论；
- 在严重或持续违规时限制参与。

## 联系与反馈入口

- 不确定该去哪里反馈：先看 [Support / 反馈与支持](SUPPORT.md)
- 不确定是不是 bug：先发 [Discussions](https://github.com/botaruibo/MyVoiceTyping/discussions)
- 明确故障：使用 [Bug report / 故障报告](https://github.com/botaruibo/MyVoiceTyping/issues/new/choose)
- 安全 / 隐私问题：先看 [Security Policy](SECURITY.md)

感谢你帮助 MyVoiceTyping 成为一个更稳、更透明、更尊重用户数据边界的中文语音输入项目。
